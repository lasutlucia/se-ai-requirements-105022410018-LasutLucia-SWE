## Reflection
AI helped organize the requirements engineering process into reusable and repeatable steps. It was useful for creating structured drafts, identifying requirement candidates, writing user stories, and building traceability links.

However, human review remains necessary because AI can produce unsupported assumptions, vague requirements, duplicate statements, or incomplete acceptance criteria. The final requirement baseline must be checked by the student to ensure that every requirement is clear, testable, measurable, and traceable to stakeholder needs.
